ğŸ”§ 2. What is Debugging?

Debugging is the process of identifying, analyzing, and fixing the errors found during testing or program execution.

Purpose

  • Solve the defect
  • Understand the cause of the error
  • Ensure software runs without failures

When does debugging happen?

  • After a failed test
  • When a program crashes
  • Unexpected or wrong output appears

ğŸ” Debugging Steps

  • Identify the bug
  • Reproduce the error
  • Locate the source line
  • Analyze the cause
  • Fix the code
  • Retest
  • Confirm the fix
