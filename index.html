Overview

Testing and debugging are complementary activities in software development. Testing finds problems; debugging finds and fixes their causes. Use testing to prove correctness; use debugging to repair defects.

Speaker note: Start the talk by asking: "Who has seen a program crash?" Use that to motivate testing (prevention) and debugging (cure).

Definitions

Debugging

What Finding, analyzing and fixing bugs (errors) in code.

Common tools Debugger, print/log statements, stack traces, unit tests.

Testing

What Verifying code behaves as expected via test cases.

Common tools Unit testing frameworks (pytest, JUnit), CI (GitHub Actions), Selenium.

Examples (ready for live demo)

Debugging example (Python)

# Bug: division by zero

def divide(a, b):
    return a / b

print(divide(10, 0))

# Fix:
def divide(a, b):
    if b == 0:
        return "Error: cannot divide by zero"
    return a / b

Testing example (table of test cases)

# Unit-style tests (pseudo)
# test_divide_normal -> divide(10,2) == 5
# test_divide_zero -> divide(10,0) == "Error: cannot divide by zero"
# test_divide_negative -> divide(-4,2) == -2
Speaker note: Run the buggy code live, let it crash, then show the fixed version and re-run. Invite students to suggest test cases before fixing.

Types of Testing & Types of Bugs

  • Unit testing — smallest parts (functions/classes)
  • Integration testing — combined modules work together
  • System testing — whole app end-to-end
  • Acceptance testing — user/requirements validation

Bug types: Syntax, Runtime, Logical, Semantic, Performance, Security.

Usage & Applications

Where you see these in industry:

  • CI/CD pipelines — run tests automatically on each push
  • Automated regression suites — prevent re-introducing bugs
  • Performance & load testing — ensure reliability
  • Security testing — protect user data
Speaker note: Mention famous tools: pytest, JUnit, Selenium, GitHub Actions, Jenkins, JMeter.

Real-time Analogy

Testing is like checking everything after a car service — test brakes, lights, and take a short drive. Debugging is like the mechanic diagnosing and replacing a faulty part after the test drive shows an issue.

Ask students for their own analogies to make the talk interactive.

Interactive Class Activities

  1. Pair up: one writes a function, the other writes tests.
  2. Bug hunt: give a file with intentional bugs, first to fix wins a small prize.
  3. Regression relay: add a feature, write tests, run CI locally (or workflow demo).

Deploy to Netlify (connect via GitHub)

Quick steps to publish this single-file site:

  1. Create a new GitHub repository (public or private).
  2. Add this file as index.html at the repository root and commit & push.
  3. Open Netlify → New site → Import from Git → choose GitHub and authorize.
  4. Select your repo. Netlify will detect a static site. Leave build command blank and set publish directory to the repository root (or simply accept defaults).
  5. Click deploy — Netlify will build and give a live URL. Use the site URL in your presentation or share it with classmates.

Alternative: Use Netlify "Drag and drop" deploy (site > Deploys > drag index.html onto the page) for a one-off quick publish.
