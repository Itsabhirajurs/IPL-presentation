üéØ 1. What is Testing?

Definition

Testing is the process of executing a program to find errors, check if the output is correct, and ensure the software behaves as expected.

Purpose of Testing

  • Detect defects before deployment
  • Improve software quality
  • Ensure performance, accuracy, and correctness
  • Check if features meet requirements

When does testing happen?

  • Before launching the software
  • After each update
  • During development (Unit Testing)

üîß 2. What is Debugging?

Definition

Debugging is the process of identifying, analyzing, and fixing the errors found during testing or during program execution.

Purpose

  • Solve the defect, not just detect it
  • Understand the cause of the error
  • Make the software run without failures

When does debugging happen?

  • After a failed test
  • When a program crashes
  • When unexpected output appears

üîç Debugging Steps

  • Identify the bug
  • Reproduce the error
  • Locate the source line
  • Analyze the cause
  • Fix the code
  • Retest
  • Confirm the fix
